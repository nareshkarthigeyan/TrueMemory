## Dlaczego TrueMemory

**Znajduje sygnal w szumie.** Twoja AI widzi tysiace wiadomosci. TrueMemory ustala, ktore z nich naprawde sie licza, i odrzuca reszte. Bez recznego tagowania, bez prompt engineeringu. Po prostu wie.

**Staje sie ostrzejszy z czasem.** To nie jest statyczna baza danych. To zywa pamiec, ktora rosnie razem z Toba. Rozwiazuje sprzecznosci, gdy zmieniasz zdanie, aktualizuje przestarzale fakty i konsoliduje to, co wie. Im dluzej uzywasz, tym lepiej dziala.

**Dziala bez Twojego myslenia o tym.** TrueMemory automatycznie przechwytuje wspomnienia z Twoich rozmow i automatycznie wstrzykuje odpowiednie do nastepnej sesji. Nigdy nie musisz recznie niczego zapisywac ani wyszukiwac. To po prostu sie dzieje.

**Jest w 100% lokalny.** Jeden plik SQLite na Twoim komputerze. Nic nie opuszcza Twojego urzadzenia. Bez chmury, bez potrzeby kluczy API. Twoje dane naleza do Ciebie.

> **Bez TrueMemory:** "Jakiego frameworka uzywamy?" Pytane po raz 12. w tym tygodniu. Twoj agent zaczyna kazda sesje z amnezja. Nie zna Twojego imienia, Twojego stacku ani niczego, co mu wczoraj powiedziales.
>
> **Z TrueMemory:** Twoj agent juz wie, ze uzywasz FastAPI, wolisz Pydantic v2 i ze Twoj auth middleware znajduje sie w `src/auth/`. Pamięta Twoje poprawki, preferencje i decyzje. W kazdej sesji, na zawsze.

## Szybki start

### Dla Claude Code / Claude CLI / Cursor / Codex CLI / Gemini CLI

```bash
curl -LsSf https://raw.githubusercontent.com/buildingjoshbetter/TrueMemory/main/install.sh | sh
```

> Instaluje wszystko w izolowanym srodowisku. Pobiera ~1,5 GB modeli AI. Zadne dane nie opuszczaja Twojego komputera. Nie wymaga sudo.

<details><summary>Nowy w terminalu? Kliknij tutaj, aby uzyskac instrukcje krok po kroku.</summary>

1. **Otworz terminal:** Mac: `Cmd + Space`, wpisz `Terminal`. Linux: `Ctrl + Alt + T`. Windows: otworz PowerShell.
2. **Wklej powyzsze polecenie** i nacisnij Enter.
3. **Poczekaj 3-5 minut** na pobranie modeli.
4. **Zamknij calkowicie swoje narzedzie AI** i otworz je ponownie (Mac: `Cmd+Q`).
5. **Wpisz "Set up TrueMemory"** i wybierz poziom.

**Windows (PowerShell):**
```powershell
irm https://raw.githubusercontent.com/buildingjoshbetter/TrueMemory/main/install.ps1 | iex
```

</details>

To wszystko. TrueMemory od teraz automatycznie zapamietuje Twoje rozmowy.

## Poziomy

Ta sama architektura, trzy poziomy. Wszystkie zawarte w jednej instalacji. Przelaczaj w dowolnym momencie mowiac "switch to Pro" lub "switch to Base".

| | Edge | Base | Pro |
|---|------|------|-----|
| **LoCoMo** | 89.6% | 92.0% | **93.0%** |
| **LongMemEval** | — | — | **92.0%** |
| **BEAM-1M** | — | — | **76.6% (SOTA)** |
| **Model Embedding** | 8 MB lekki | 600 MB wysoka dokladnosc | 600 MB wysoka dokladnosc |
| **Reranker** | 22M parametrow | 149M parametrow | 149M parametrow |
| **HyDE search** | — | — | ✅ (wymaga klucza LLM API) |
| **Dziala na** | Kazdym komputerze, tylko CPU | 4 GB+ RAM | 4 GB+ RAM + klucz API |

**Edge** dziala wszedzie. **Base** to najsilniejszy calkowicie offline'owy poziom. **Pro** dodaje rozszerzanie zapytan oparte na AI dla najwyzszych wynikow.

## Python API

```python
from truememory import Memory

m = Memory()

m.add("Prefers dark mode and TypeScript", user_id="alex")
m.add("Works at Anthropic as a senior engineer", user_id="alex")

results = m.search("What are Alex's preferences?", user_id="alex")
results = m.search_deep("career history?", user_id="alex")  # wielorundowe, wyzsza dokladnosc
```

| Metoda | Opis |
|--------|-------------|
| `m.add(content, user_id)` | Zapisz wspomnienie |
| `m.search(query, user_id)` | Wyszukaj (potok 6-warstwowy + reranker) |
| `m.search_deep(query, user_id)` | Wielorundowe wyszukiwanie agentowe |
| `m.get(id)` / `m.get_all(user_id)` | Pobierz wspomnienia |
| `m.update(id, content)` / `m.delete(id)` | Zmodyfikuj lub usun |
| `m.stats()` | Statystyki systemu |

[Pelna dokumentacja API ->](../docs/python-api.md)

## FAQ

<details><summary><strong>Gdzie przechowywane sa moje dane? Czy cos jest wysylane do chmury?</strong></summary>

Wszystko jest przechowywane lokalnie w `~/.truememory/memories.db`. Poziomy Edge i Base nie wykonuja zadnych zewnetrznych wywolan. Pro wysyla jedynie tekst zapytania do LLM w celu rozszerzenia zapytania. Twoje wspomnienia nigdy nie sa przesylane.
</details>

<details><summary><strong>Czy potrzebuje zainstalowanego Pythona?</strong></summary>

Nie. Instalator uzywa [uv](https://docs.astral.sh/uv/) do zarzadzania sandboxowanym Pythonem 3.12. Twoj systemowy Python nie jest modyfikowany.
</details>

<details><summary><strong>Dlaczego nie uzyc po prostu wiekszego okna kontekstowego?</strong></summary>

Okna kontekstowe sa drogie, wolne i puste na poczatku kazdej sesji. TrueMemory zapewnia natychmiastowe przywolywanie przy zerowej liczbie tokenow kontekstu, w czasie ponizej 200ms.
</details>

<details><summary><strong>Czy TrueMemory zbiera telemetrie?</strong></summary>

Anonimowa telemetria uzytkowania (wywolania narzedzi, liczba sesji, informacje o platformie) jest domyslnie wlaczona. **Nigdy** nie sledzimy tresci wspomnien, zapytan, sciezek plikow ani kluczy API. Rezygnacja: `export TRUEMEMORY_TELEMETRY=off`
</details>
